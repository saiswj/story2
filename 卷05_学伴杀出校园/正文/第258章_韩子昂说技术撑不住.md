# 第258章：韩子昂说技术撑不住

韩子昂把技术评审放在晚上十点。

不是因为他喜欢熬夜。

而是白天的项目教室太吵。

社群消息、合作电话、用户退款、马成海的嗓门，都让他没法把一件事讲完整。

晚上十点，灯只开了一半。

大屏上没有增长曲线。

只有一张很丑的系统链路图。

用户进来。

选择学校。

匹配课程。

购买资料。

进入反馈。

触发更新。

客服回复。

退款或续费。

每一条线旁边，都被韩子昂标了红点。

马成海看得头皮发麻。

“你这是产品流程，还是犯罪现场？”

韩子昂没理他。

“如果只扩本校和三个外校，红点可以靠人工盯。”

他把图放大。

“如果八周二十所，红点会叠在一起。到时候不是哪个地方慢，是我们根本不知道哪个地方先炸。”

陆远坐在第一排。

林夏坐在他旁边，手里拿着用户反馈分类表。

韩子昂点开第一组风险。

“学校名称和课程版本。”

他放出一条昨晚的误匹配记录。

江北财大一名学生选了“财务管理”，系统默认推了本校旧版资料。幸好付费前被适用范围页拦住，才没有成交。

“现在靠人工复核能挡。”

韩子昂看向陆远。

“学校一多，人工复核挡不住。”

第二组风险。

客服。

他调出三段聊天。

同样一句“能不能退款”，在不同场景下答案完全不一样。

没开放付费的学校，可以直接劝退。

适用范围错误的订单，应该优先退款。

用户买完又不想学，那是普通售后。

“如果我们招人，但不给权限和判断树，客服会乱答。”

第三组风险。

更新。

资料版本一多，哪份更新推给谁，谁有权修改，修改后谁复核，都会变成问题。

韩子昂把日志翻到最后。

“现在很多东西是我手动改。”

马成海试探着问：“手动改不行吗？”

韩子昂终于看他。

“你希望我一天二十四小时不睡，还是希望用户等我睡醒？”

马成海闭嘴。

林夏低声说：“继续。”

韩子昂点开第四组。

监控。

这次是服务器。

绿色曲线看起来还好，但右侧有几次尖峰。

“这几次不是正常访问，是云习社上线后，有人批量抓我们的适用范围页。”

陆远眉头一动。

“能确认来源吗？”

“不能确定是谁，但访问节奏不像普通用户。”

马成海立刻炸了。

“周正阳？”

陆远没有顺着骂。

“先按异常访问处理。”

韩子昂把担心技术撑不编号补上，点了点头。

“我已经加了限制，但这只是临时补丁。扩张后，数据和权限都要重做。不然对手不用打价格战，光抓我们的适配结果，就能省掉很多成本。”

这句让屋里安静了。

之前大家担心的是服务器崩。

现在才发现，技术承载不只是能不能访问。

还包括他们辛辛苦苦跑出来的学校适配、用户反馈、课程差异，会不会被别人拿去当捷径。

韩子昂关掉图，打开最后一页。

【最低可承接方案】

一，学校入口分级。

二，课程版本库先建最小闭环。

三，客服工单必须留痕。

四，资料更新权限拆分。

五，异常访问监控。

六，所有新学校先跑反馈志愿者，不直接大规模付费。

马成海看着六条，声音低了很多。

“这要多久？”

韩子昂说：“最快十天。”

“十天？”

“这是最快。”

马成海一拳砸在自己大腿上。

“十天够云习社铺多少群了。”

韩子昂也急。

只是他的急不靠嗓门。

“那你告诉我，出错后谁来承担？”

马成海抬头。

韩子昂说：“用户不会骂投资人，不会骂窗口期，只会骂学伴骗子。”

这句比所有报错都重。

陆远看着大屏。

他想起自己刚重来时，系统给出的那些简单建议。

不送早餐。

先报名。

拿成绩说话。

那时候的选择清楚得像单选题。

现在不是。

快有快的道理。

稳有稳的代价。

林夏把用户反馈表推到韩子昂面前。

“你的技术方案里，加一个用户提示。”

韩子昂问：“什么提示？”

“每个学校入口都显示状态。已适配、试点中、暂不建议购买。不要让用户猜。”

韩子昂想了想，点头。

“可以。”

马成海忽然说：“那能不能让试点中的学校先预约？不直接付费，但留下需求。”

林夏隔着报错表看他。

“这句可以。”

马成海松了口气。

他不是不懂稳定。

他只是怕大家一讲风险，就把增长全按住。

陆远终于开口。

“明天给秦川的方案，就按这个逻辑。”

他在白板上写：

【不是不扩，是分级扩。】

韩子昂看着那行字，肩膀稍微放松了一点。

“我今晚先把入口状态改出来。”

陆远说：“别通宵。”

韩子昂已经把电脑拉到面前。

“说晚了。”

项目教室里几个人都笑了一下。

笑完又各自低头。

凌晨零点四十，第一版“学校状态”入口跑通。

已适配是绿色。

试点中是黄色。

暂不建议购买是灰色。

用户点进灰色入口，会先看到一句话：

【当前资料差异较大，不建议直接购买。可提交课程版本，等待适配通知。】

林夏看完，说：“像学伴该说的话。”

韩子昂没抬头。

“别夸，我还没测完。”

凌晨一点，监控页忽然跳出一条新告警。

异常访问再次出现。

这一次，对方抓的不是价格页。

而是刚刚上线不到二十分钟的学校状态接口。

韩子昂的脸色沉下去。

马成海看着那串访问记录，终于明白技术压力已经不只是内部问题。

有人在外面等着他们一动。

陆远站起身。

“先封，再留证据。”

韩子昂手指飞快敲键盘。

林夏把告警截图拖进复盘文档。

马成海盯着屏幕，低声骂了一句。

“这仗，真开始变脏了。”

韩子昂把报警阈值往下调了一档，又把最容易拥堵的入口单独列出来。技术不是后台的事，一旦崩在用户面前，前面所有增长都会变成笑话。

马成海盯着技术风险备忘录很久，最后把标题改成处理说明。这份技术压力说明不提气，但每一项都能追到负责人。

技术报警页面上，接口超时记录一条接一条往下刷。

韩子昂盯着屏幕，声音比平时更短：“不是小抖动。再冲一波会炸。”

马成海嘴唇动了动，没敢催“再撑一下”。

韩子昂把外校入口临时加上排队提示，又把晚上十点后的高峰时段限流写进公告。页面没说漂亮话，只告诉用户：当前排队人数、预计进入时间、失败后自动保留名额。

第一条反馈很快弹出来：【这回至少知道不是我手机坏了。那超时怎么办？】

陆远让刘鹏回：【超时不扣次数，队列保留。今晚十点后我们限流，明早补扩容结果。】

十分钟后，投诉群少了一条转发帖。

群里的火气没有全消，可问题终于不再像黑箱。马成海看着那一列排队数字，没再开玩笑。

当天晚上，外校入口晚上十点后限流，投诉少了一半。韩子昂把扩容账单贴到电脑边，低声说：“要花钱。别再拿热度赌机器。”
